১. সিস্টেম ওয়ার্কফ্লো (System Workflow)

ব্যবহারকারী কীভাবে সিস্টেমে কাজ করবে এবং ডাটা কীভাবে প্রসেস হবে তার ফ্লো নিচে বর্ণনা করা হলো:

  1. ওয়েবসাইট ভিজিট (Client Browser): একজন ভিজিটর যখন ই-কমার্স সাইটে ঢুকে পেজভিউ বা কেনাকাটা (Purchase) করে, তখন জাভাস্ক্রিপ্ট ট্রিগার হয়।
  2. ফার্স্ট-পার্টি ডোমেইন (CNAME Setup): ডাটা সরাসরি আপনার মূল ট্র্যাকিং সার্ভারে না গিয়ে ক্লায়েন্টের সাব-ডোমেইনে (যেমন: trk.client-store.com) যায়। এটি অ্যাডব্লকার (AdBlocker) এবং iOS ট্র্যাকিং রেস্ট্রিকশন বাইপাস করতে সাহায্য করে।
  3. লারাভেল API ইনেজেশন (Ingestion API): এনজিনিক্স (Nginx) এবং লারাভেল খুব দ্রুত ডাটা গ্রহণ করে। এখানে রেট লিমিট এবং API কী চেক করা হয়।
  4. কিউতে পাঠানো (Redis Queue): ডাটাগুলো সাথে সাথে ডাটাবেসে সেভ করে ব্যাকগ্রাউন্ড প্রসেসের জন্য Redis Queue তে পাঠানো হয় এবং ওয়েবসাইটকে ২00 OK রেসপন্স দিয়ে দেওয়া হয়।
  5. লারাভেল ওয়ার্কার (Laravel Workers): ব্যাকগ্রাউন্ডে সুপারভাইজার (Supervisor) চালিত ওয়ার্কারগুলো কিউ থেকে ডাটা বের করে আনে।
  6. এক্সটার্নাল API তে সেন্ড: ওয়ার্কারগুলো ডাটা গুলোকে ফেসবুক কনভার্সন API (Facebook CAPI) এবং গুগল অ্যানালিটিক্স ৪ (GA4) এ পাঠিয়ে দেয়।
  7. লগিং ও ড্যাশবোর্ড (Dashboard logs): সকল সাকসেস বা ফেইলর লগ আকারে সেভ হয়, যা ক্লায়েন্ট তার ড্যাশবোর্ডে দেখতে পারে।

২. অ্যাডভান্সড ফিচারসমূহ (Advanced System Features)

ইভেন্ট ডিডুপ্লিকেশন (Event Deduplication)

যদি কোনো ইউজার থ্যাংক ইউ পেজ (Thank You Page) দুইবার রিলোড দেয়, তবে একই Purchase ইভেন্ট দুইবার যেতে পারে। এটি আটকাতে আমরা event_id ব্যবহার করেছি। লারাভেল API চেক করবে এই আইডিটি ইতিমধ্যে ডাটাবেসে আছে কিনা, থাকলে স্কিপ করবে। ফেইসবুকও এই event_id দিয়ে ডাবল ডাটা ফিল্টার করে নিবে।

রেট লিমিটিং (Rate Limiting)

ডিডিওএস (DDoS) বা ফেক ট্রাফিক আটকাতে লারাভেলের throttle:300,1 মিডলওয়্যার ব্যবহার করা হয়েছে। অর্থাৎ, প্রতি মিনিটে কোনো ক্লায়েন্ট ৩০০ টির বেশি ইভেন্ট পাঠাতে পারবে না।

৩. প্রোডাকশন ডিপ্লয়মেন্ট (Production Deployment - Ubuntu VPS)

DigitalOcean বা AWS এ Ubuntu 22.04 LTS সার্ভার ব্যবহার করুন।

Nginx ও অন্যান্য প্যাকেজ ইনস্টল:

sudo apt update && sudo apt upgrade -y
sudo apt install nginx mysql-server redis-server supervisor php8.3-fpm php-mysql php-redis -y

Nginx CNAME কনফিগারেশন:

আপনার ক্লায়েন্টদের কাস্টম ডোমেইন গুলো সাপোর্ট করার জন্য Wildcard কনফিগার করতে হবে।

server {
  listen 80;
  server_name tracking-saas.com *.tracking-saas.com;
  root /var/www/tracking-saas/public;
  index index.php;

  location / {
    try_files $uri $uri/ /index.php?$query_string;
  }
}

Supervisor কিউ (Queue) সেটআপ:

ব্যাকগ্রাউন্ডে ইভেন্ট সেন্ড করার জন্য ওয়ার্কার চালু রাখতে হবে। নিচের কোডটি /etc/supervisor/conf.d/tracking.conf এ লিখুন:

[program:tracking-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/tracking-saas/artisan queue:work redis --sleep=3 --tries=3
autostart=true
autorestart=true
user=www-data
numprocs=8 # ৮টি প্যারালাল ওয়ার্কার

তারপর রান করুন: sudo supervisorctl update

৪. সিস্টেম স্কেলিং - ১০০০+ ক্লায়েন্ট (Scaling Strategy)

  • MySQL এর বদলে ClickHouse: যখন দিনে ১০ মিলিয়নের বেশি ইভেন্ট আসবে, তখন MySQL ধীর গতির হয়ে যাবে। তখন events টেবিলের জন্য 'ClickHouse' ডাটাবেস ব্যবহার করতে হবে।
  • Laravel Octane: লারাভেলকে মেমোরিতে ধরে রেখে প্রতি সেকেন্ডে ৩০০০+ রিকোয়েস্ট প্রসেস করতে FrankenPHP বা Swoole চালিত Octane ব্যবহার করুন।
  • আলাদা Redis সার্ভার: মূল সার্ভারের চাপ কমাতে Redis এর জন্য AWS ElastiCache বা আলাদা ছোট VPS সার্ভার নিন।

৫. সাবস্ক্রিপশন ও ব্যবসা (Monetization System)

আপনার SaaS প্ল্যাটফর্ম থেকে আয় করার জন্য মিটারড বিলিং (Metered Billing) ব্যবহার করতে পারেন।

  • ফ্রি প্ল্যান (Free Tier): মাসে ১০,০০০ টি ইভেন্ট ফ্রি।
  • পেইড প্ল্যান (Paid Tier): মাসে ১ লাখ ইভেন্টের জন্য স্টার্টার প্যাক ২০ ডলার।

Redis এ কাউন্টার ব্যবহার করে খুব সহজেই ক্লায়েন্টের মাসিক ইভেন্ট ট্র্যাক করতে পারবেন। লিমিট পার হলে সিস্টেম আটোমেটিকভাবে ইভেন্ট সেন্ড করা বন্ধ করে ক্লায়েন্টকে পেমেন্ট করতে বলবে।
